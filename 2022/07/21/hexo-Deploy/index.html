hexo-Deploy

使用Travis CI 实现 Hexo 博客自动部署

演示地址

https://littletreeden.github.io/

参考仓库

https://github.com/LittleTreeden/LittleTreeden.github.io

1. 下载工具Node.js 和 Git

  • 下载完成后使用 node -v 检测是node否安装完成

  • git --version 检测git是否安装完成

2.新建Git仓库 username.github.io

  • 在右上角点击新建仓库。

  • 仓库名使用 username.github.io，点击创建。

username 为 github 账户名

3.将仓库clone到本地

4. 下载并安装hexo

  1. 安装hexo脚手架 npm i hexo-cli -g。

检查hexo是否安装完成 hexo -v。

  1. 新建分支 hexo 。

  1. 新建一个空文件夹，并初始化hexo hexo init。

  1. 使用npm安装依赖库npm install。

  1. 使用hexo生成静态网页 hexo g。

  1. 运行hexo hexo s

​ 打开 http://localhost:4000/ 验证。

以上为hexo本地部署。

5.Travis配置

  1. 生成GitHub验证TOKEN。

https://github.com/settings/tokens

只选仓库权限即可

  1. 复制token 只会显示一次，记得复制 。在第6步会用到，记得保存。

  2. 进入Travis ，按照提示开启，新注册用户需要选择Plan，需要绑定信用卡。

  1. 对我们的仓库设置相关配置。

  2. 添加token环境变量，将github生成的token填入，变量名为GH_TOKEN。

7.将本地仓库切换至hexo，将刚才的博客源码添加到我们的仓库中，并删除public 和 node_modules（生成的静态文件和依赖库文件）。

  1. 添加.travis.yml文件，并修改相应配置（可直接复制，用户名和email需要修改）。
  1. 修改_config.yml 的deploy节。
  1. 修改_config.yml将URL重新设定为自己的仓库路径。
  1. 执行命令 git push origin hexo 提交代码。
